# ブロック図（block-beta）

> ⚠️ **beta構文**: 表示環境によってレンダリングされない場合があります。使用前に確認してください。

## 概要

グリッドベースでブロック（コンポーネント）を配置し、システム構成を表現する図。レイアウトをより細かくコントロールできる。

## 使いどころ

- システムの物理的・論理的なコンポーネント配置
- レイアウト指定が必要なアーキテクチャ図
- 画面レイアウトの概略（ワイヤーフレーム的用途）

## 使わないケース

- フローや順序が重要 → `flowchart`
- サービス間の依存・通信 → `architecture-beta`

---

## 基本テンプレート

```
block-beta
    columns 3
    A["ブロックA"]:2
    B["ブロックB"]
    C["ブロックC"]:3
    A --> C
```

`columns N` でグリッドの列数を指定。`:N` でブロックの幅（列数）を指定。

---

## 実例

### 例1: レイヤードアーキテクチャ

```mermaid
block-beta
    columns 3
    P["プレゼンテーション層"]:3
    A["アプリケーション層"]:3
    D1["注文ドメイン"] D2["在庫ドメイン"] D3["決済ドメイン"]
    I["インフラストラクチャ層"]:3
    P --> A
    A --> D1
    A --> D2
    A --> D3
    D1 --> I
    D2 --> I
    D3 --> I
```

### 例2: マイクロサービス構成

```mermaid
block-beta
    columns 3
    GW["APIゲートウェイ"]:3
    OS["注文サービス"] IS["在庫サービス"] PS["決済サービス"]
    DB1[("注文DB")] DB2[("在庫DB")] DB3[("決済DB")]
    GW --> OS
    GW --> IS
    GW --> PS
    OS --> DB1
    IS --> DB2
    PS --> DB3
```
