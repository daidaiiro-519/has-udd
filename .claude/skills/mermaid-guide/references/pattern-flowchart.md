# フローチャート（flowchart）

## 概要

ノードと矢印で構成される汎用の関係・フロー図。Mermaidで最も多用する構文。方向によって階層図・プロセス図・関係図を使い分ける。

## 使いどころ

- **TD（上→下）**: 階層・包含関係、ツリー構造、組織図
- **LR（左→右）**: プロセス・変換フロー、役割・関係図、依存の流れ

## 使わないケース

- アクター間のメッセージ順序 → `sequenceDiagram`
- クラス・継承関係 → `classDiagram`
- エンティティの多重度 → `erDiagram`
- 状態遷移 → `stateDiagram-v2`

---

## 基本テンプレート

```mermaid
flowchart LR
    A[ノードA] --> B[ノードB]
    B --> C[ノードC]
```

---

## ノードの形

| 記法 | 形 | 用途 |
|---|---|---|
| `[text]` | 四角形 | 一般的なノード |
| `(text)` | 角丸四角形 | 開始・終了・状態 |
| `((text))` | 円形 | イベント・ポイント |
| `{text}` | ひし形 | 分岐・条件（判断フローはテキスト形式を推奨） |
| `[(text)]` | 円柱 | データベース・ストレージ |
| `[/text/]` | 平行四辺形 | 入出力 |
| `[[text]]` | 二重四角形 | サブルーチン |

## 矢印の種類

| 記法 | 見た目 | 用途 |
|---|---|---|
| `-->` | 実線矢印 | 通常の関係・フロー |
| `---` | 実線（矢印なし） | 関連・接続 |
| `-.->` | 点線矢印 | 非同期・弱い依存 |
| `==>` | 太線矢印 | 強調・主要フロー |
| `--\|ラベル\|-->` | ラベル付き矢印 | 関係の説明 |

## 実例

### 例1: 階層・包含関係（TD）

```mermaid
flowchart TD
    BD[ビジネスドメイン]
    BD --> SD1[中核の業務領域]
    BD --> SD2[一般的な業務領域]
    BD --> SD3[補完的な業務領域]
```

### 例2: プロセス・変換フロー（LR）

```mermaid
flowchart LR
    BK[業務知識] -->|発見| KM[概念モデル]
    KM -->|設計| RM[解決モデル]
    RM -->|実装| SC[ソースコード]
```

### 例3: 役割・関係図（LR）+ subgraph

```mermaid
flowchart LR
    BE[業務エキスパート] --> YA[要求分析の担当者]
    YA --> YS[要求仕様書]
    YS --> KA[開発者]
    KA --> CODE["&lt;/コード&gt;"]

    subgraph 翻訳の連鎖
        YA
        YS
    end
```

### 例4: 細分化・分解（LR）

```mermaid
flowchart LR
    A[顧客サービス] -->|細分化| B[案件の担当者選び\n🔴 中核]
    A -->|細分化| C[ヘルプデスク\n🔵 一般]
    A -->|細分化| D[架電システム\n🔵 一般]
    A -->|細分化| E[シフト管理\n⚪ 補完]
```

---

## subgraphの使い方

```mermaid
flowchart LR
    A --> B
    B --> C

    subgraph グループ名
        B
        C
    end
```

subgraphは関連するノードをまとめてグルーピングするときに使う。境界づけられたコンテキストや業務領域の境界の表現に有効。
