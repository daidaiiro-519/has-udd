# ユーザージャーニー（journey）

## 概要

ユーザーが目標を達成するまでの体験の流れを、各ステップのスコア（満足度）とともに表現する図。

## 使いどころ

- ユーザー体験（UX）の可視化
- 業務フローの「つらさ」の見える化
- 改善ポイントの特定

## 使わないケース

- システム間の処理順序 → `sequenceDiagram`
- 状態の変化 → `stateDiagram-v2`

---

## 基本テンプレート

```mermaid
journey
    title ジャーニーのタイトル
    section セクション名
        タスク名: スコア: アクター名
```

スコアは `1`（最低）〜`5`（最高）で指定する。

---

## 実例

### 例1: ヘルプデスク担当者のジャーニー

```mermaid
journey
    title 新規案件対応のジャーニー
    section 案件受付
        メール確認: 3: 担当者
        案件を登録: 2: 担当者
        優先度を判断: 3: 担当者
    section 対応
        担当者にアサイン: 4: リーダー
        顧客に回答: 4: 担当者
        解決確認: 5: 担当者
    section クローズ
        案件をクローズ: 4: 担当者
        レポート作成: 2: 担当者
```

### 例2: 複数アクターの比較

```mermaid
journey
    title オンライン購入体験
    section 商品選択
        商品を検索: 3: 顧客
        詳細を確認: 4: 顧客
        カートに追加: 5: 顧客
    section 購入
        決済情報を入力: 2: 顧客
        注文確定: 5: 顧客
        在庫確認: 3: 倉庫担当
    section 配送
        梱包: 4: 倉庫担当
        発送通知: 5: システム
```
