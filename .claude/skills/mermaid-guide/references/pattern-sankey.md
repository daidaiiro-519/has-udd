# サンキー図（sankey-beta）

> ⚠️ **beta構文**: 表示環境によってレンダリングされない場合があります。使用前に確認してください。

## 概要

フローの量（ボリューム）を線の太さで表現する図。リソースの流れ・配分・変換過程を量的に可視化する。

## 使いどころ

- リソース・予算・工数の流れと配分
- トラフィックの経路と量
- 変換プロセスでの損失・変化の可視化

## 使わないケース

- 量が不要な関係図 → `flowchart`
- 単純な構成比 → `pie`

---

## 基本テンプレート

```
sankey-beta
    ソース,ターゲット,値
```

CSV形式で「ソースノード, ターゲットノード, 量」を記述する。

---

## 実例

### 例1: 開発工数の配分

```mermaid
sankey-beta
    開発工数,中核ドメイン,60
    開発工数,補完ドメイン,25
    開発工数,一般ドメイン,15
    中核ドメイン,設計,20
    中核ドメイン,実装,30
    中核ドメイン,テスト,10
```

### 例2: ユーザーのサービス利用フロー

```mermaid
sankey-beta
    訪問者,会員登録,30
    訪問者,離脱,70
    会員登録,初回購入,20
    会員登録,未購入,10
    初回購入,リピーター,15
    初回購入,離脱,5
```
