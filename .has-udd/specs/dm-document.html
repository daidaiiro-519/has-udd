dm-document

目的

Document（構造化された成果物）のライフサイクルと一貫性を表す集約。状態は生成→検証→描画の順にのみ進み、後続版で置換される。

集約ルート

ルート
Document
識別子
DocumentId

不変条件

  • status は CREATED → VALIDATED → RENDERED → SUPERSEDED の順にのみ進み、逆行しない
  • schemaRef は常に存在し、content が schema に適合しない限り VALIDATED へ進めない
  • SUPERSEDED は終端状態であり、以後どのコマンドも受け付けない

属性

属性

value が等しければ等価

コマンド

create

schemaRef と documentId から schema に適合する骨格を生成し、CREATED 状態で作る。

引数

引数意味
schemaRef適用する schema
documentId新しい識別子

発行イベント

  • DocumentCreated

validate

content が schema に適合するか検証する（副作用なし）。適合すれば VALIDATED へ進める。

発行イベント

  • DocumentValidated

render

VALIDATED な Document を x-render に従い成果物に描画し、宣言された配置先（各ツール）へ反映して RENDERED にする。

発行イベント

  • DocumentRendered
  • DocumentDeployed

supersede

後続版に置き換え、SUPERSEDED にする。

引数

引数意味
successorId後継 Document の識別子

発行イベント

  • DocumentSuperseded

ユニットテスト仕様

Feature: Document 集約の不変条件

  Scenario: status は逆行できない
    Given RENDERED 状態の Document
    When validate へ戻そうとする
    Then 状態遷移は拒否され、状態は RENDERED のままである

  Scenario: 未検証では render できない
    Given CREATED 状態の Document
    When render する
    Then 拒否され、状態は CREATED のままである

  Scenario: SUPERSEDED は終端
    Given SUPERSEDED 状態の Document
    When 任意のコマンドを実行する
    Then 拒否される
