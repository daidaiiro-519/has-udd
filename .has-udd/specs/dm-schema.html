dm-schema

目的

Schema（Document の型定義）の不変条件とバージョン進化を表す集約。Document が機械生成・検証・描画できることを保証し、版の移行を司る。

集約ルート

ルート
Schema
識別子
SchemaRef

不変条件

  • 値フィールドに oneOf / anyOf を持たない（scaffold が骨格を機械生成できる）
  • content の各ブロックは additionalProperties を閉じる（固定 properties のみ）
  • 再帰は有界である（無限ネストを許さない）
  • 各ブロックの x-render は RenderMetaSchema の閉じた語彙にのみ従う
  • status の enum は遷移順に並び、先頭が初期状態である
  • 公開済みの版は遡って構造を変えない（後方互換を壊さない）

Version

属性

属性意味
value単調増加する版識別子（例: v1, v2）

等価判定

value が等しければ等価

コマンド

publishVersion

新しい版の Schema を公開し、以後その版で Document を作れるようにする。scaffoldability 不変条件を満たすことが前提。

引数

引数意味
version公開する版

発行イベント

  • SchemaVersionPublished

deprecateVersion

古い版を非推奨にし、新規 Document の作成を止める（既存は移行まで有効）。

引数

引数意味
version非推奨にする版

migrateDocuments

既存 Document を旧版から新版へ移行する。版を上げる方向にのみ行う。

引数

引数意味
fromVersion移行元の版
toVersion移行先の版

発行イベント

ユニットテスト仕様

Feature: Schema 集約の不変条件

  Scenario: 値フィールドに oneOf を持てない
    Given 値フィールドに oneOf を含む Schema
    When scaffoldability を検証する
    Then scaffold 不能として拒否される

  Scenario: 公開済みの版は後方互換を壊さない
    Given PUBLISHED の Schema 版
    When 既存ブロックに必須フィールドを追加しようとする
    Then 後方互換を壊す変更として拒否される

  Scenario: 移行は版を上げる方向にのみ行う
    Given v1 と v2 の Schema
    When v2 から v1 へ移行しようとする
    Then 拒否される
