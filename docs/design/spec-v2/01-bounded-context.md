# 受注コンテキスト（Ordering）

EC サイトの受注業務を扱う境界づけられたコンテキスト。カートの確定から配送手配までの一貫性を保つ。決済・在庫・配送は別コンテキストと連係する。

## ユビキタス言語

この境界の内側でのみ通用する語彙。

| 用語 | 定義 |
|---|---|
| 注文（Order） | 顧客が確定した購入の単位。明細・金額・配送先・状態を持つ |
| 注文明細（OrderLine） | 注文に含まれる商品ごとの行。商品・数量・単価を持つ |
| 在庫引当（Allocation） | 注文確定時に在庫を確保すること |
| 配送先（ShippingAddress） | 商品を届ける住所 |
| 金額（Money） | 通貨つきの金額。異なる通貨は加算できない |
| 顧客（Customer） | 注文を行う主体（このコンテキストでは ID で参照する外部集約） |

## 構成要素

| 種別 | メンバー |
|---|---|
| subdomain | 受注管理（中核）／決済（一般）／配送通知（補完） |
| aggregate | 注文（Order）／カート（Cart） |
| usecase | 注文を確定する／注文をキャンセルする／支払いを受け付ける |

## 文脈マップ

| 相手コンテキスト | 関係 | 内容 |
|---|---|---|
| 決済コンテキスト | 顧客／供給（こちらが顧客） | 支払い結果を受け取り、注文を PAID に進める |
| 在庫コンテキスト | パートナー | 在庫引当を相互に調整する |
| 配送コンテキスト | 腐敗防止層 | 外部配送業者の API を自前モデルに翻訳して扱う |
