# 受注管理（subdomain）

注文の正確な受付と在庫引当を担う、受注コンテキストの中核業務領域。複雑な業務ルール（引当・割引・キャンセル可否）を持ち、競争優位の源泉となる。

## カテゴリー

**中核（core）** — 注文処理の正確さと在庫引当の最適化が他社との差別化を生む。複雑で進化し続けるため、社内開発し厚く設計する。

## 所属ユースケース

- 注文を確定する（uc-place-order）
- 注文をキャンセルする（uc-cancel-order）

## 実装ガイド

ドメインモデル（集約）で実装する。ポートとアダプターを採用し、業務ロジックを永続化・外部連係から独立させる。テストはピラミッド形（単体重視）。

## ドメインサービス

単一の集約に属さない、複数集約に跨るステートレスな業務ロジック。

| サービス | 責務 |
|---|---|
| 在庫引当サービス（StockAllocationService） | 注文集約と在庫集約に跨り、注文明細に対して引当可能かを判定し在庫を確保する |
| 送料計算サービス（ShippingFeeService） | 配送先と注文重量から送料を算出する（複数のポリシーを統合） |
